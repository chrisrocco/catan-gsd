1. **Wire ROLL_DICE into actions.ts:**
   - Import `applyRollDice` and `applyDiscard` from `'./resources.js'`
   - Add `case 'ROLL_DICE': return applyRollDice(state, action);` to the switch statement
   - Add `case 'DISCARD_RESOURCES': return applyDiscard(state, action);` to the switch statement
   - IMPORTANT: `applyDiscard` checks `discardQueue[0]` not `activePlayer` for the discard turn. During the `discard` phase, the engine's turn-order check (`action.playerId !== state.activePlayer`) will reject non-active-player discards. Since `applyRollDice` does NOT update `activePlayer` to `discardQueue[0]`, the discard phase needs the active player to be the first discarding player. Fix: In `applyRollDice`, when entering discard phase, set `activePlayer: discardQueue[0]` (the first player who needs to discard). In `applyDiscard`, when advancing to next discardQueue entry, set `activePlayer: newDiscardQueue[0]` (or keep current activePlayer if queue empty and transitioning to robber-move — set `activePlayer` back to the original roller). BUT we don't have the original roller stored. Simpler approach: In `applyDiscard`, set `activePlayer` to `newDiscardQueue[0]` when queue is not empty, or to the current `activePlayer` from the ORIGINAL state when transitioning to `robber-move` (since after discard the roller moves robber). Actually the simplest correct fix: modify `applyRollDice` to store `activePlayer: discardQueue[0]` when roll is 7 with discards. Then `applyDiscard` should set `activePlayer: newDiscardQueue[0]` if queue still has entries, otherwise keep the original roller as activePlayer for robber-move. The roller is still `state.activePlayer` at the time `applyRollDice` is called — but after that transition, `activePlayer` becomes the first discarder. After ALL discards, who moves the robber? The ROLLER. So we need to track the roller. The simplest approach: store the roller's ID. But GameState doesn't have a `rollerId` field. Alternative: look at `applyDiscard` — when `newDiscardQueue` is empty, who should be `activePlayer` for `robber-move`? It should be the player whose turn it is (the roller). We can find this by looking at `state.playerOrder` and `state.turnNumber`. Actually, the simplest fix without changing GameState: DON'T change activePlayer in applyRollDice — instead, modify the discard path in applyAction to SKIP the turn-order check for DISCARD_RESOURCES. Add a special case before the turn-order check: if `state.phase === 'discard' && action.type === 'DISCARD_RESOURCES'`, skip the `activePlayer` check and let `applyDiscard` validate via `discardQueue[0]`. This is the least invasive fix.

   Final approach (least invasive): In `applyAction`, add a special case BEFORE the turn-order check:
   ```typescript
   // Discard phase: any player in discardQueue may discard, not just activePlayer
   if (state.phase === 'discard' && action.type === 'DISCARD_RESOURCES') {
     return applyDiscard(state, action);
   }
   ```
   This bypasses both turn-order and phase-legality checks (applyDiscard does its own validation via discardQueue[0]).

Create `packages/server/src/bot/scoring.ts` with the following functions:

1. **TOKEN_PIPS constant:** `Record<number, number>` mapping number tokens to pip counts: `{2:1, 3:2, 4:3, 5:4, 6:5, 8:5, 9:4, 10:3, 11:2, 12:1}`

2. **scoreVertex(state, vertexKey):** Score a vertex for settlement desirability.
   - Sum pip counts from adjacent hexes (skip desert, skip hexes with no resource/number)
   - Multiply pip total by 10 (primary factor)
   - Add diversity bonus: `resourcesSeen.size * 2` (max 3 distinct resources for a corner)
   - Add port bonus: `vertex.port ? 1 : 0` (small tiebreaker)
   - Return total score

3. **legalSetupVertices(state):** Find all vertices legal for setup placement.
   - Filter `Object.keys(state.board.vertices)` where vertex has no building AND no adjacent vertex has a building (distance rule)
   - Also exclude vertices that have no adjacent land hexes (sea-only vertices are useless)

4. **pickWeightedTop(candidates, scoreFn, rand, topN=3):** Weighted random selection from top-N.
   - Score all candidates, sort descending, take top N
   - Use weighted random: `threshold = rand() * totalScore`, iterate and subtract
   - Throw if no candidates

5. **computeVisibleVP(state, playerId):** Count VP for a player.
   - Count settlements (1 VP) and cities (2 VP) from `state.board.vertices`
   - Add 2 if `state.longestRoadHolder === playerId`
   - Add 2 if `state.largestArmyHolder === playerId`
   - Add `player.vpDevCards`

6. **findLeader(state, excludePlayerId):** Find highest-VP opponent.

7. **chooseBestRobberHex(state, botId):** Select optimal robber hex.
   - Find leader via `findLeader`
   - Score each hex: pips + 20 if leader has building on hex
   - Skip current robber hex and desert
   - Must not select a hex where only the bot itself has buildings
   - Fallback: any non-current, non-desert hex

8. **computeBuildGoal(state, botId):** Determine what the bot should build next.
   - Priority order: city (if has settlement to upgrade and approaching resources), settlement (if has reachable vertex), road (if expanding toward target vertex), dev-card (if can afford and deck not empty)
   - Return `'city' | 'settlement' | 'road' | 'dev-card' | null`

9. **chooseTrade(state, botId):** Decide whether to execute a bank/port trade.
   - Determine build goal via `computeBuildGoal`
   - Check if trading excess resource at best rate could acquire a needed resource
   - Proactive 7-card avoidance: if total cards >= 6 and can't build, trade most-excess for least-held
   - Return `Action | null`
