  <action>
  1. Update `packages/client/src/store/gameStore.ts` to add interaction state:
     - `pendingAction: 'settlement' | 'road' | 'city' | 'robber' | null` — what the player is trying to place
     - `setPendingAction(action)` — sets pendingAction
     - `clearPendingAction()` — sets to null

  2. Update `packages/client/src/components/board/HexBoard.tsx` to add interactive placement:

     **Compute valid placements based on pendingAction and game phase:**

     When `pendingAction === 'settlement'` OR phase is `setup-forward`/`setup-reverse`:
     - Find valid vertices: iterate board.vertices, filter where:
       - No building exists (`vertex.building === null`)
       - No adjacent vertex has a building (check `vertex.adjacentVertexKeys` → look up each, check building)
       - In setup phases: any empty isolated vertex is valid
       - In post-roll: must be adjacent to own road (check `vertex.adjacentEdgeKeys` → look up edges, check road.playerId)
     - Render glowing dots at valid vertex positions:
       - Circle with `r=8`, fill semi-transparent player color, CSS animation pulse
       - Use Tailwind-compatible SVG animation: `<animate>` element for opacity pulse or className
       - onClick: emit `submitAction({ type: 'PLACE_SETTLEMENT', playerId: '', vertexKey: v.key })`
       - Then clearPendingAction()

     When `pendingAction === 'road'` OR phase is `setup-forward`/`setup-reverse` (after settlement placed, i.e., setupPlacementsDone is odd):
     - Find valid edges: iterate board.edges, filter where:
       - No road exists (`edge.road === null`)
       - At least one endpoint vertex has own building OR adjacent edge has own road
       - In setup phases: must be adjacent to the settlement just placed
         (simplification: any edge touching an own building vertex with no road)
     - Render glowing lines at valid edge positions:
       - Thick semi-transparent line with pulse animation
       - onClick: emit `submitAction({ type: 'PLACE_ROAD', playerId: '', edgeKey: e.key })`
       - Then clearPendingAction()

     When `pendingAction === 'city'`:
     - Find valid vertices where player has a settlement (building.type === 'settlement' && building.playerId === localPlayerId)
     - Render highlight circles on those vertices
     - onClick: emit `submitAction({ type: 'UPGRADE_CITY', playerId: '', vertexKey: v.key })`

     When `pendingAction === 'robber'` OR phase is `robber-move`:
     - Highlight all hex centers EXCEPT current robber hex
     - onClick on hex: emit `submitAction({ type: 'MOVE_ROBBER', playerId: '', hexKey: hex.key })`

     **Auto-trigger placement mode:**
     - In setup phases (`setup-forward`, `setup-reverse`): auto-set pendingAction based on setupPlacementsDone
       - Even: show settlement placement
       - Odd: show road placement
     - In `robber-move` phase: auto-set pendingAction to 'robber'

     **Key UX decisions (from CONTEXT.md):**
     - Click to place — no drag and drop
     - No confirmation step — click places immediately
     - Invalid clicks do nothing (locations without highlights are not clickable)
  </action>

  1. Create `packages/client/src/components/hud/ActionBar.tsx`:
     Context-sensitive buttons based on game phase and active player.
     Read gameState, playerId, pendingAction from store.

     Show buttons based on phase (only when it's the local player's turn, i.e., `gameState.activePlayer === playerId`):

     **pre-roll phase:**
     - "Roll Dice" button → emits `submitAction({ type: 'ROLL_DICE', playerId: '' })`
     - "Play Dev Card" dropdown (if player has unplayed dev cards and devCardsPlayedThisTurn === 0)

     **post-roll phase:**
     - "Build Settlement" → sets pendingAction to 'settlement' (if player can afford: check hand vs BUILD_COSTS)
     - "Build Road" → sets pendingAction to 'road' (if affordable)
     - "Upgrade City" → sets pendingAction to 'city' (if affordable and has settlements)
     - "Buy Dev Card" → emits `submitAction({ type: 'BUY_DEV_CARD', playerId: '' })` (if affordable)
     - "Trade" → opens TradePanel
     - "Play Dev Card" dropdown (if has unplayable dev cards and devCardsPlayedThisTurn === 0 and no devCardBoughtThisTurn for that card type — actually the server validates this, just show button if has cards)
     - "End Turn" → emits `submitAction({ type: 'END_TURN', playerId: '' })`

     **robber-steal phase:**
     - Show buttons for each opponent with a building on the robber hex
     - "Steal from {player}" → emits `submitAction({ type: 'STEAL_RESOURCE', playerId: '', targetPlayerId: target })`
     - "Skip" button if no targets → emits `submitAction({ type: 'SKIP_STEAL', playerId: '' })`

     **road-building phase:**
     - Show "Place Road ({roadsLeft} remaining)" text
     - Road placement is handled by HexBoard highlights

     **year-of-plenty phase:**
     - Show resource selection: 5 resource buttons, player clicks 2
     - Track selections in local state
     - On second click: emit `submitAction({ type: 'PLAY_DEV_CARD', playerId: '', card: 'year-of-plenty', yearOfPlentyResources: [r1, r2] })`

     **Dev card play sub-actions:**
     - Knight: emit play dev card, then enters robber-move phase automatically
     - Monopoly: show resource type selector, on click emit with monopolyResource
     - Road Building: emit play dev card, enters road-building phase automatically
     - Year of Plenty: handled above

     Disabled states: gray out buttons when player can't afford or conditions not met.
     Styled: horizontal bar above the player hand panel, with icon buttons.
